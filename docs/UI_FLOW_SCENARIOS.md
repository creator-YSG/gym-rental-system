## 시나리오 A: 이용권 없는 회원

**회원 정보**
- 전화번호: `01011111111`
- 이름: 이용권없음
- 이용권: 없음
- 특징: 로그인은 되지만 대여 불가

### 1️⃣ 홈 화면 (초기 상태)

**📸 스크린샷 필요: `A_01_home_initial.png`**

**화면 구성:**
- 상단: "운동복 대여" 타이틀
- 중앙: 전화번호 입력 필드 (플레이스홀더: "010-0000-0000")
- 하단: 숫자 키패드 (1-9, 0, 삭제, C)
- 최하단: 로그인 버튼 (비활성화 상태, 회색)

**설명:**
시스템 첫 화면입니다. 사용자는 전화번호를 입력하여 로그인합니다.

---

### 2️⃣ 홈 화면 (전화번호 입력 중)

**📸 스크린샷 필요: `A_02_home_input_partial.png`**

**화면 변화:**
- 전화번호 입력 필드: "010-1111-1" (6자리 입력)
- 로그인 버튼: 여전히 비활성화 (11자리 미만)

**설명:**
전화번호를 입력하는 중입니다. 11자리가 입력되어야 로그인 버튼이 활성화됩니다.

---

### 3️⃣ 홈 화면 (전화번호 입력 완료)

**📸 스크린샷 필요: `A_03_home_input_complete.png`**

**화면 변화:**
- 전화번호 입력 필드: "010-1111-1111" (11자리 완료)
- 로그인 버튼: **활성화** (청록색)

**설명:**
전화번호 11자리 입력 완료. 로그인 버튼을 클릭할 수 있습니다.

---

### 4️⃣ 홈 화면 (로딩 중)

**📸 스크린샷 필요: `A_04_home_loading.png`**

**화면 변화:**
- 반투명 오버레이
- 중앙: 로딩 스피너
- 텍스트: "회원 정보 확인 중..."

**설명:**
로그인 버튼 클릭 후 API 호출 중입니다. 회원 정보를 서버에서 확인하고 있습니다.

---

### 5️⃣ 상품 선택 화면 (이용권 없음)

**📸 스크린샷 필요: `A_05_rental_no_payment.png`**

**화면 구성:**
- 상단 헤더:
  - 좌측: "이용권없음님" 이름 표시
  - 우측: "로그아웃" 버튼
  - **중요**: 잔액 정보 → "이용권 없음" 또는 "사용 가능한 이용권 없음"
- 카테고리 탭: 상의, 하의, 수건, 땀수건
- 상품 그리드: 각 상품 카드 (사이즈, 이름, 가격, 재고)
- 하단: 장바구니 영역 (빈 상태)

**설명:**
로그인 성공 후 상품 선택 화면입니다. 하지만 이 회원은 이용권이 없어서 대여를 진행할 수 없습니다.

---

### 6️⃣ 마이페이지 (빈 상태)

**📸 스크린샷 필요: `A_06_mypage_empty.png`**

**화면 구성:**
- 우측에서 슬라이드되어 나온 패널
- 헤더: "← 내 카드" (뒤로가기 버튼)
- 구독권 섹션: "구독권이 없습니다." (회색 텍스트)
- 금액권 섹션: "금액권이 없습니다." (회색 텍스트)

**설명:**
"내 카드" 버튼을 클릭하면 나타나는 마이페이지입니다. 이 회원은 이용권이 전혀 없습니다.

---

### ⚠️ 대여 시도 시

**설명:**
상품을 장바구니에 담고 "대여하기" 버튼을 클릭하면:
- **에러 메시지** 또는
- **결제 수단 선택 모달**이 뜨지만 선택할 수단이 없음

**결과:** 대여 불가

### 6️⃣ 마이페이지 (구독권만)

  - 카드 아이템:
    - 상태: "✅ 사용 중" (초록색)
    - 이름: "1개월 기본 이용권"
    - 정보: "2024-12-01 ~ 2024-12-31"

- **금액권 섹션:**

**설명:**
구독권만 보유한 회원의 마이페이지입니다.

### 7️⃣ 상품 선택 후 장바구니

**📸 스크린샷 필요: `B_07_cart_one_item.png`**

**화면 변화:**
- 상품 그리드: "상의 105" 카드에 "1개 선택" 배지
- **하단 장바구니:**
  - 장바구니 헤더: "장바구니" | "1개 **1,000원**"
  - 장바구니 아이템:
    - "운동복 상의 105" | 수량 조절 버튼 (-/+) | "1,000원"
  - **대여하기 버튼: 활성화** (청록색)

**설명:**
상품을 장바구니에 담았습니다. 구독권이 있으므로 대여가 가능합니다.

---

### 8️⃣ 비밀번호 입력 모달 (전체화면)

**📸 스크린샷 필요: `B_08_password_modal.png`**

**화면 구성:**
- **전체화면 모달** (배경 어두움)
- 중앙:
  - 제목: "결제 비밀번호" (큰 글씨)
  - 비밀번호 입력 표시: 6개의 원 (빈 상태)
  - 숫자 키패드: 1-9, 0, C, ← (3x4 그리드)
  - 하단: "취소" / "확인" 버튼

**설명:**
"대여하기" 버튼 클릭 시 **결제 모달 없이** 바로 비밀번호 입력 화면으로 이동합니다. (구독권 1개만 있어서 자동 배정)

---

### 9️⃣ 비밀번호 입력 중

**📸 스크린샷 필요: `B_09_password_input_partial.png`**

**화면 변화:**
- 비밀번호 입력 표시: 3개의 원이 채워짐 (청록색)
- 나머지 3개는 비어있음

**설명:**
6자리 비밀번호 중 3자리를 입력한 상태입니다.

---

### 🔟 비밀번호 입력 완료

**📸 스크린샷 필요: `B_10_password_complete.png`**

**화면 변화:**
- 비밀번호 입력 표시: 6개의 원 모두 채워짐
- "확인" 버튼: 활성화 (청록색)

**설명:**
비밀번호 6자리 입력 완료. "확인" 버튼을 클릭하면 대여가 실행됩니다.

---

### 1️⃣1️⃣ 대여 완료 화면

**📸 스크린샷 필요: `B_11_complete.png`**

**화면 구성:**
- 중앙 카드:
  - 상단: 체크마크 아이콘 (원형, 청록색)
  - 제목: "대여 완료!"
  - 날짜/시간 표시
  - **대여 상품 목록:**
    - "운동복 상의 105" | "1개"
  - **결제 정보:**
    - "📋 구독권 사용"
    - "상의 1회" (초록색 텍스트)
  - **총 결제 금액:** "0원" (구독권 사용)
  - 하단: "물품이 곧 나옵니다. 잠시만 기다려주세요."
  - 카운트다운: "**5**초 후 처음 화면으로"

**설명:**
대여가 완료되었습니다. 구독권으로 결제되어 금액은 0원이고, 구독권 사용 횟수가 차감됩니다.
